**문서 버전:** 1.0
**작성일:** 2026-05-26
**상태:** 초안 (Draft)

---

## 1. 개요 (Overview)

### 1.1 서비스 한 줄 정의

사용자의 자격증·어학성적·전공을 입력하면, 전국 공기업 가산점 DB를 분석해 합격 가능성이 가장 높은 공기업을 순위별로 추천해주는 웹서비스.

### 1.2 배경 및 문제 정의

공기업 취업 준비생은 지원 전 각 기업별 가산점 항목을 일일이 채용공고에서 찾아 비교해야 한다. 이 과정은 다음과 같은 문제를 야기한다.

- **정보 분산**: 가산점 기준이 공기업마다 상이하고 채용공고·내규·국가법령 등 여러 소스에 분산되어 있다.
- **비교 불가**: 수십~수백 개 공기업을 개인이 수동 비교하기 현실적으로 불가능하다.
- **기회 손실**: 자신에게 유리한 가산점 항목이 있음에도 몰라서 지원하지 않는 경우가 발생한다.
- **정보 비대칭**: 취업 컨설팅 기관은 이 정보를 유료로 제공하지만 일반 구직자는 접근이 어렵다.

### 1.3 솔루션 요약

1. 사용자가 보유 자격증·어학성적·졸업학과를 프로필에 등록한다.
2. 서비스가 전국 공기업 가산점 DB와 매칭 분석을 수행한다.
3. 가산점 합산 점수 및 항목 적합도를 기준으로 유리한 공기업 순위를 제공한다.

## 4. 핵심 기능 요구사항 (Feature Requirements)

### 4.1 사용자 프로필 등록

#### 4.1.1 자격증 등록

- **국가기술자격**: 기사, 산업기사, 기능사 등 등급별 입력
  - 예: 정보처리기사, 전기기사, 컴퓨터활용능력 1급/2급 등
- **국가전문자격**: 공인노무사, 세무사, 회계사 등
- **공인어학자격**: 별도 어학 섹션으로 분리
- 자격증 검색 자동완성 기능 (한국산업인력공단 자격 목록 기반)
- 취득 연도 입력 (유효기간이 있는 자격증 처리)

#### 4.1.2 어학 성적 등록

| 시험 | 입력 항목 |
|------|---------|
| TOEIC (토익) | 점수, 취득일 |
| TOEIC Speaking (토스) | 등급 (Lv.6~8), 점수, 취득일 |
| OPIc (오픽) | 등급 (IM1~AL), 취득일 |
| TOEFL | 점수, 취득일 |
| IELTS | 점수, 취득일 |
| JPT / JLPT | 점수/급수, 취득일 |
| HSK | 급수, 취득일 |
| 기타 | 직접 입력 |

- 성적 유효기간 자동 만료 처리 (시험별 유효기간 기준 적용)

#### 4.1.3 학력 및 전공 등록

- 최종 학력 (고졸, 전문학사, 학사, 석사, 박사)
- 졸업 학과 / 전공 계열
  - 대분류: 이공계, 상경계, 인문사회계, 사범계, 예체능 등
  - 소분류: 전기전자, 기계, 화학, 컴퓨터공학, 경영, 경제 등
- 학점 (선택 입력, GPA 4.5 / 4.3 / 100점 만점 변환 지원)
- 복수전공 / 부전공 입력 가능

#### 4.1.4 기타 가산 항목

- 취업 지원 대상 (보훈) 여부
- 장애인 등록 여부
- 지역인재 (지방대학 졸업 여부 및 지역)
- 비수도권 거주 여부

### 4.2 공기업 가산점 DB

#### 4.2.1 DB 구성 항목

각 공기업별로 다음 정보를 구조화하여 저장:

- **기업 기본 정보**: 기업명, 유형(공기업/준정부기관/기타공공기관), 주무부처, 본사 소재지
- **가산점 항목**: 자격증별 가산점 비율(%), 어학성적별 기준 및 가산점, 전공 우대 여부, 특별 가산 항목
- **채용 정보 메타**: 최근 채용 공고 URL, 마지막 DB 업데이트 일자
- **적용 기준**: NCS 직무, 직렬, 전형 단계 (서류/필기/면접)

#### 4.2.2 DB 관리 정책

- 공공기관 경영정보 공개시스템(알리오), 각 기관 채용공고, 나라일터 등을 정기적으로 크롤링·파싱
- 분기별 1회 이상 전체 DB 검증 업데이트
- 사용자 신고 기능: 잘못된 가산점 정보에 대한 제보 접수

---

### 4.3 가산점 매칭 및 추천 엔진

#### 4.3.1 매칭 로직

1. 사용자 프로필의 각 항목(자격증, 어학성적, 전공)을 공기업 가산점 기준과 1:1 매핑
2. 항목별 가산점 점수를 합산하여 **총 가산점 예상 점수** 산출
3. 가산점 외 전공 적합도, 보훈/장애/지역인재 가점을 별도 반영
4. 기업별 **가산점 점수** 및 **적합 항목 수**를 기준으로 순위 정렬

#### 4.3.2 추천 결과 출력

- **추천 목록 (기본)**: 상위 20개 공기업 카드 형태 표시
  - 기업명, 유형, 총 예상 가산점, 해당 항목 요약
- **상세 페이지**: 특정 기업 선택 시 항목별 가산점 내역 및 공고 링크
- **필터 옵션**: 지역, 공기업 유형, 직무 계열로 필터링
- **정렬 옵션**: 가산점 높은 순 / 항목 적합도 순 / 최근 채용 공고 순

#### 4.3.3 "갭 분석" 기능 (Gap Analysis)

- 현재 프로필 기준 TOP 추천 기업 대비 **추가 취득 시 상승 가능 가산점** 표시
- 예: "OPIc IM2 → IH 취득 시 A공사에서 +2점 상승"
- 효율적인 다음 스펙 추천 (취득 난이도 대비 가산점 이득)

## 6. 정보 구조 및 주요 화면 흐름 (IA & User Flow)

```
[랜딩 페이지]
    ↓
[회원가입 / 로그인]
    ↓
[프로필 등록 마법사]
    ├── Step 1: 학력·전공 입력
    ├── Step 2: 보유 자격증 추가

         └── 갭 분석 (추가 취득 시 변화)
    ↓
[마이페이지]
    ├── 프로필 수정
    ├── 관심 기업 목록
    └── 알림 설정
```

## 7. 데이터 모델 개요 (Data Model Overview)

### 사용자 (User)
```
user_id, email, name, created_at, last_login
```

### 프로필 (Profile)
```
profile_id, user_id, profile_name,
education_level, major_category, major_detail, school_region,
is_veterans, is_disabled, is_local_talent
```

### 자격증 항목 (UserCertificate)
```
id, profile_id, certificate_id (FK), acquired_date, grade
```

### 어학 성적 (UserLanguageScore)
```
id, profile_id, exam_type, score, grade, acquired_date, expiry_date
```

### 공기업 (PublicEnterprise)
```
enterprise_id, name, type, ministry, location, last_updated
```

### 가산점 기준 (BonusPointRule)
```
rule_id, enterprise_id, category (자격증/어학/전공/기타),
item_id, condition_detail, bonus_point_percentage, source_url, updated_at
```

## 8. 기술 스택 (Tech Stack)

| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | Next.js (React), TypeScript, Tailwind CSS |
| 백엔드 | Next.js API Routes (서버리스 함수) |
| 데이터베이스 | Supabase (PostgreSQL) — 테이블, RLS, Realtime |
| 캐싱 | Supabase Edge Functions + Next.js ISR/캐시 헤더 |
| 배포 | Vercel (프론트엔드 + API Routes 통합 배포) |
| 크롤링/DB 갱신 | Python (Scrapy / BeautifulSoup) + Supabase REST API로 적재 |
| 인증 | Supabase Auth (이메일, 카카오, 네이버, 구글 OAuth) |
| 파일 스토리지 | Supabase Storage |
| 알림 | Supabase Edge Functions + Resend (이메일) |

### 8.1 아키텍처 선택 이유

**Vercel**
- Next.js와 완벽 통합으로 프론트엔드·API를 단일 레포에서 배포 가능
- 서버리스 함수로 별도 백엔드 서버 불필요 → 초기 운영 비용 최소화
- 자동 CDN, 프리뷰 배포, CI/CD 파이프라인 내장

**Supabase**
- PostgreSQL 기반으로 복잡한 가산점 매칭 쿼리(JOIN, 집계) 처리에 적합
- Row Level Security(RLS)로 사용자별 데이터 접근 제어 내장
- Supabase Auth로 OAuth 소셜 로그인 및 세션 관리 일원화
- 관리형 서비스로 DB 운영 부담 최소화, 무료 티어로 MVP 개발 가능

### 8.2 환경 구성

| 환경 | 설명 |
|------|------|
| Development | 로컬 + Supabase 로컬 CLI (`supabase start`) |
| Staging | Vercel Preview 배포 + Supabase 스테이징 프로젝트 |
| Production | Vercel Production + Supabase 프로덕션 프로젝트 |

## 10. 리스크 및 고려사항 (Risks & Considerations)

| 리스크 | 설명 | 대응 방안 |
|--------|------|----------|
| 데이터 정확성 | 공기업 가산점 기준이 채용마다 변경될 수 있음 | 정기 업데이트 + 출처 명시 + 사용자 신고 기능 |
| 법적 책임 | 잘못된 가산점 정보로 인한 불이익 | 서비스 이용약관에 면책 조항 명시, 정보 제공 목적 고지 |
| DB 구축 공수 | 전국 공기업 수백 개의 가산점 기준 수집 | Phase 1에서 주요 공기업 100개로 시작, 크롤링 자동화로 점진 확장 |
| 개인정보 | 보훈·장애 등 민감 정보 수집 | 최소 수집 원칙, 별도 암호화, 선택 입력 처리 |
| 경쟁 서비스 | 유사 서비스 등장 가능성 | DB 커버리지와 정확성으로 차별화, 빠른 시장 선점 |

## 11. 성공의 정의 (Definition of Done)

MVP 완성 기준:
- 사용자가 프로필 등록 후 5분 이내에 추천 결과를 확인할 수 있다.
- 전국 주요 공기업 100개 이상에 대한 가산점 데이터가 서비스에 반영되어 있다.
- 각 추천 결과에 항목별 가산점 상세 내역과 출처가 표시된다.
- 서비스가 모바일에서도 정상적으로 사용 가능하다.
